# 社区法治风险智能预警与精准普法平台项目描述

## 一、项目定位

本项目面向基层社会治理与法治建设场景，建设“社区法治风险智能预警与精准普法平台”。平台以北京市西城区街道治理为示范场景，围绕案件线索接入、风险识别、智能预警、任务分派、处置反馈、效果评估和精准普法，形成一套可展示、可运行、可扩展的法治风险闭环治理系统。

平台不局限于单一欠薪场景，而是覆盖邻里纠纷、劳动争议、合同诈骗、物业纠纷、未成年人保护、老年人权益保护、消费维权、工程款纠纷等多类社区高发法治风险。系统通过结构化案件数据、风险规则、NLP 分析、大模型生成能力和可视化驾驶舱，辅助基层治理部门更早发现风险、更准分派任务、更清晰评估治理成效。

## 二、建设目标

1. 建立统一的数据接入能力，将案件线索、风险事件、街道信息、主体信息、证据材料和处置结果纳入统一管理。
2. 建立社区法治风险识别机制，自动识别高发案件类型、重点主体、重点区域和风险等级。
3. 建立风险分级预警机制，将风险划分为红、橙、黄、蓝等等级，并在驾驶舱中直观展示。
4. 建立全流程闭环管理机制，实现从风险发现、智能预警、任务分派、处置反馈到效果评估的完整链路。
5. 建立精准普法内容引擎，根据街道风险画像和案件类型自动生成治理建议与普法方案。
6. 建立评审展示友好的数据驾驶舱，突出平台的业务价值、技术能力和治理效果。

## 三、核心业务流程

平台的核心链路为：

```text
数据接入 -> 风险发现 -> 智能预警 -> 任务分派 -> 处置反馈 -> 效果评估 -> 精准普法
```

各环节含义如下：

| 环节 | 说明 |
|---|---|
| 数据接入 | 导入社区案件、线索、主体、街道、金额、人数、证据、时间等信息 |
| 风险发现 | 通过案件类型、金额、频次、主体重复出现、街道聚集度等指标识别潜在风险 |
| 智能预警 | 根据风险规则和画像结果形成预警等级，并推送到驾驶舱实时预警列表 |
| 任务分派 | 对高风险预警生成治理任务，自动带入街道、风险等级、高发类型、承办单位和处置建议 |
| 处置反馈 | 承办单位反馈核查情况、联动处置结果和阶段性治理动作 |
| 效果评估 | 从风险变化、事件变化、治理效果、普法效果四类指标生成处置成效评估卡 |
| 精准普法 | 根据风险画像生成面向不同街道、不同群体、不同案件类型的普法内容 |

## 四、主要功能模块

### 1. 数据驾驶舱

驾驶舱是平台的第一展示入口，承担评审时的整体视觉呈现和态势总览功能。主要包括：

- 中央区域：街道风险热力图，按街道或点位展示社区法治风险等级。
- 左侧面板：核心指标概览、案件类型分布、月度趋势。
- 右侧面板：实时预警列表、高风险主体 TOP5、普法推送效果。
- 底部链路：风险发现、已预警、已分派、处置中、已反馈、已评估的闭环进度。

驾驶舱强调“看得见风险、点得进详情、派得出任务、评得出效果”。

### 2. 社区风险治理

社区风险治理模块以西城区各街道为入口，支持按街道查看案件列表、风险画像和治理建议。主要能力包括：

- 展示西城区全部街道。
- 点击街道后查看该街道相关案件。
- 点击案件后查看案件详情和闭环轨迹。
- 生成街道社区法治风险画像。
- 识别高发案件类型和重点风险因素。
- 生成治理建议与精准普法方案。

该模块用于体现平台从“单案处理”向“区域治理”的升级。

### 3. 智能预警与任务分派

实时预警列表展示最新触发的风险线索。对需要处置的预警，平台支持一键进入任务创建页，并自动带入：

- 所属街道
- 风险等级
- 高发案件类型
- 建议承办单位
- 建议处置动作
- 预警标题与关联案件

任务创建后进入闭环任务表，支持按阶段查看风险发现、已预警、已分派、处置中、已反馈、已评估等任务。

### 4. 闭环轨迹与效果评估

每个案件详情页和任务详情页均展示闭环轨迹，用环形时间轴呈现案件当前所处阶段。当前阶段之前的内容正常展示，尚未到达的阶段不提前展示详情，保证业务逻辑清晰。

任务进入“已评估”后，平台自动生成处置成效评估卡，评估维度包括：

- 风险变化：风险等级是否下降、风险评分是否下降。
- 事件变化：同类案件新增是否减少、重复主体是否再次触发。
- 治理效果：任务按期办结率、联动处置完成率。
- 普法效果：推送覆盖人数、阅读率、参与率、回访问卷反馈。

该能力用于突出平台不是停留在“发现风险”，而是能够证明“治理是否有效”。

### 5. NLP 研判与普法内容引擎

平台支持对案件文本进行自然语言处理，提取案件类型、主体、地点、金额、证据、风险特征等信息，并结合规则和大模型能力生成：

- 风险识别结果
- 街道风险画像
- 治理建议
- 普法主题
- 普法文案
- 面向不同群体的个性化普法方案

大模型接口可通过硅基流动 API 配置，未配置时可使用本地规则生成兜底结果，保证系统演示稳定。

## 五、技术架构

项目采用前后端分离架构：

```text
justice-risk-platform/
├── frontend/      React + TypeScript + Vite + Ant Design
├── backend/       FastAPI 主后端
├── mock_data/     示例案件与演示数据
├── scripts/       数据生成、导入和初始化脚本
├── docs/          技术文档、维护文档、项目材料
└── deploy/        部署相关配置
```

### 前端技术栈

- React
- TypeScript
- Vite
- Ant Design
- AntV L7
- ECharts

前端负责驾驶舱、街道风险治理、案件详情、任务创建、任务列表、风险画像和普法内容展示。

### 后端技术栈

- FastAPI
- SQLAlchemy
- SQLite 或可替换关系型数据库
- 模块化业务目录
- 大模型 API 接入
- 本地规则兜底能力

后端负责案件数据管理、风险分析、驾驶舱聚合、街道画像、闭环任务保存、任务状态流转和效果评估生成。

## 六、数据设计概述

平台数据主要包括：

- 案件语料数据：案件编号、标题、类型、当事人、主体、街道、金额、人数、案情摘要、证据材料、风险特征。
- 街道治理数据：街道名称、经纬度、风险等级、案件数量、高发类型。
- 预警数据：预警标题、风险等级、触发原因、建议处置单位、建议动作。
- 闭环任务数据：任务名称、关联预警、街道、风险等级、承办单位、办理期限、处置动作、当前阶段、反馈、评估结果。
- 普法数据：普法主题、覆盖人数、阅读率、参与率、回访满意度。

## 七、项目创新点

1. 从单一案件办理转向社区法治风险治理，突出区域画像和系统治理。
2. 从静态展示转向闭环管理，覆盖发现、预警、派单、反馈和评估。
3. 从人工经验判断转向智能研判，利用 NLP 和大模型辅助风险识别。
4. 从统一普法转向精准普法，根据街道、群体和案件类型生成差异化内容。
5. 从结果汇总转向效果证明，通过处置成效评估卡量化治理成效。
6. 从原型展示转向可运行系统，具备真实接口、任务保存和数据导入链路。

## 八、应用场景

平台可用于以下场景：

- 基层检察机关开展社区法治风险研判。
- 街道、社区、网格治理部门发现高发风险。
- 对重点主体、重点区域、重点类型案件进行预警。
- 对风险事件进行任务分派和闭环跟踪。
- 面向老年人、劳动者、未成年人、消费者等群体开展精准普法。
- 在项目评审、比赛答辩、申报书和 PPT 中展示数字法治建设成果。

## 九、演示说明

推荐演示顺序如下：

1. 打开数据驾驶舱，展示全区风险热力图和核心指标。
2. 点击实时预警，查看高风险案件详情。
3. 从预警进入任务创建页，展示自动带入的分派信息。
4. 打开闭环任务列表，查看不同阶段任务。
5. 进入案件详情，展示环形闭环轨迹。
6. 将任务推进到已评估，展示处置成效评估卡。
7. 进入社区风险治理，选择街道并生成风险画像。
8. 展示治理建议和精准普法方案。

## 十、项目价值

本项目的价值在于把基层法治治理中的“线索多、风险散、分派慢、反馈弱、评估难、普法泛”转化为可计算、可预警、可派单、可跟踪、可评估、可展示的数字化流程。

平台既能服务评审展示，也具备继续扩展为实际业务系统的基础。后续可继续接入真实案件库、法律知识图谱、政策文件库、短信或微信公众号普法渠道、统一身份认证和多部门协同系统，进一步形成完整的基层法治风险治理平台。
